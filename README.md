# django-webapp
Разработка нейронных сетей. Веб-приложение классификации изображении.

---

## Описание

Проект для выполнения домашнего задания №1 по курсу "Распознавание природных сигналов".

**Задача:** Веб-приложение для классификации изображений трёх классов рептилий:
- 🦎 Варан
- 🦎 Хамелеон
- 🦎 Геккон

**Модель:** ResNet20 (ONNX, точность ~68%)

---

## Быстрый старт

### Локальный запуск (без S3)

```bash
# 1. Перейдите в папку проекта
cd django-webapp/web-site-dl

# 2. Установите зависимости
pip install -r ../../requirements-django.txt

# 3. Запустите Django
python manage.py runserver
```

**Сервисы после запуска:**
- 🌐 Django: `http://localhost:8000`

## Модель

### ResNet20
- **Архитектура:** 20 слоёв (3×3 BasicBlock + FC)
- **Параметров:** ~272K
- **Вход:** 32×32×3 RGB
- **Точность:** 85.14% (Adam, lr=0.001, 30 эпох, очищенный датасет)
- **Формат:** ONNX (оптимизированный)

### Доступные модели
| Модель | Optimizer | LR | Accuracy |
|--------|-----------|------|----------|
| ResNet20 | Adam | 0.001 | 85.14% |

## Веб-приложение

### Функции
- Классификация загруженных изображений
- Выбор модели из списка
- Загрузка новых ONNX моделей
- Удаление моделей
- Современный адаптивный дизайн

### Использование
1. Откройте `http://localhost:8000/`
2. Загрузите изображение (drag & drop или выбор файла)
3. Выберите модель (если несколько)
4. Нажмите "Классифицировать"
5. Получите результат

---

## Добавление новой модели

1. Экспортируйте модель в ONNX из проекта обучения
2. Откройте `http://localhost:8000/uploadModel`
3. Загрузите `.onnx` файл
4. Модель появится в списке доступных

### Требования к модели
- Формат: `.onnx`
- Вход: 32×32 RGB, нормализация [0, 1]
- Входной тензор: имя `input`, форма `[batch, 3, 32, 32]` или `[batch, 32, 32, 3]`
- Выход: логиты для 3 классов (Варан, Хамелеон, Геккон)

## Контрольные вопросы

### 1. Архитектура ResNet20

```
ResNet20:
├── Conv2d(3, 16, 3x3, padding=1) + BatchNorm + ReLU
├── Layer1: 3× BasicBlock(16, 16, stride=1)
├── Layer2: 3× BasicBlock(16, 32, stride=2)
├── Layer3: 3× BasicBlock(32, 64, stride=2)
├── Global AvgPool (8x8)
└── Linear(64, 3)
```

### 2. Аугментация данных

- RandomHorizontalFlip (p=0.5)
- RandomRotation (15°)
- ColorJitter (brightness=0.2, contrast=0.2, saturation=0.2)

### 3. Оптимизация

- **Оптимизатор:** Adam (lr=0.001, weight_decay=1e-4)
- **Scheduler:** StepLR (step=60, gamma=0.2)
- **Loss:** CrossEntropyLoss (label_smoothing=0.1)
